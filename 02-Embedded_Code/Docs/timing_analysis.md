# MyFOC 时序链路分析：从定时器到 FOC_step() 的完整时间嵌套

> 生成日期: 2026-06-15
> 适用硬件: STM32G431CBU6 (HSE = 24 MHz)
> 适用分支: main

---

## 1. 概述

本文档详细描述 MyFOC 系统中从 TIM1 定时器 → ADC 注入采样 → 中断回调 → FOC_step() → PWM 更新的完整时序链路，以及各级时钟频率、执行时间预算和嵌套关系。

### 1.1 核心数据流

```
TIM1 中心对齐 PWM (CH4 OC4REF 触发)
    │
    ├── ADC1/ADC2 注入组同步触发 (10kHz，每个 PWM 周期触发一次)
    │       │
    │       ├── ADC1: CH3(A相) → CH12(C相)  共 0.95 µs
    │       └── ADC2: CH3(B相)              共 0.475 µs
    │               │
    │               └── ADC1 JEOC 中断 (ADC1_2_IRQn, 优先级 0)
    │                       │
    │                       ├── 读取三相 ADC 值
    │                       ├── offset 修正 → A 转换
    │                       ├── FOC_step()  (核心计算, ~6-12 µs)
    │                       ├── 写入 TIM1->CCR1/2/3 (更新 PWM)
    │                       └── DMA 发送 VOFA 调试帧 (尽力, 不阻塞)
    │
    └── 主循环 (10ms 周期, 低频任务)
            ├── 处理上位机命令
            ├── 更新母线电压 (ADC2 regular 轮询)
            └── 启动/停机逻辑
```

## 2. 系统时钟树

### 2.1 时钟源配置

| 参数 | 值 |
|------|-----|
| HSE 外部晶振 | **24 MHz** (`HSE_VALUE = 24000000`, 定义于 `stm32g4xx_hal_conf.h:118`) |
| PLLM | **/3** (`RCC_PLLM_DIV3 = 0x03`, HAL 内部减 1 后写入寄存器位[7:4]=2, 硬件解码为 2+1=3) |
| PLLN | **×40** |
| PLLR | **/2** |
| **SYSCLK** | **24MHz / 3 × 40 / 2 = 160 MHz** |
| 电压调节器 | SCALE1_BOOST |
| Flash 延迟 | 4 等待周期 |

### 2.2 PLLM 寄存器编码证明

HAL 库 `stm32g4xx_hal_rcc.h:2978-2979`:
```c
#define __HAL_RCC_PLL_PLLM_CONFIG(__PLLM__) \
    MODIFY_REG(RCC->PLLCFGR, RCC_PLLCFGR_PLLM, ((__PLLM__) - 1) << RCC_PLLCFGR_PLLM_Pos)
```

`system_stm32g4xx.c:250`:
```c
pllm = ((RCC->PLLCFGR & RCC_PLLCFGR_PLLM) >> 4) + 1U;
```

所以 `RCC_PLLM_DIV3` 中 "3" 就是真正的分频系数——HAL 内部写 `(3-1)=2` 到寄存器，硬件读取 `2+1=3`。

### 2.3 总线时钟分配

| 总线 | 预分频 | 时钟频率 | 挂载外设 |
|------|--------|----------|---------|
| HCLK (AHB) | /1 | **160 MHz** | DMA, GPIO |
| PCLK1 (APB1) | /1 | **160 MHz** | TIM2, TIM4, USART3 |
| PCLK2 (APB2) | /1 | **160 MHz** | TIM1, ADC1, ADC2 |

### 2.4 TIM1 定时器时钟

APB2 预分频 = 1，根据 STM32G4 参考手册 (RM0440)：
> 当 APB 预分频 = 1 时，定时器时钟 = PCLK（不翻倍）

```
TIM1CK = PCLK2 = 160 MHz   (周期 6.25 ns)
```

### 2.5 ADC 时钟

| 参数 | 值 |
|------|-----|
| ADC12 时钟源 | SYSCLK (160 MHz) |
| 时钟预分频 | ADC_CLOCK_ASYNC_DIV4 |
| **ADC 时钟** | **160MHz / 4 = 40 MHz** |
| ADC 周期 | **25 ns** |

## 3. TIM1 PWM 定时器详解

### 3.1 寄存器配置

| 参数 | 寄存器值 | 含义 |
|------|----------|------|
| Prescaler (PSC) | 0 | 不分频，计数时钟 = TIM1CK |
| Period (ARR) | 7999 | 计数范围 0 → 7999 → 0 |
| CounterMode | CENTERALIGNED1 | 中心对齐模式 1 |
| ClockDivision (CKD) | DIV2 | tDTS = 2 × CK_INT |
| RepetitionCounter | 1 | 每 (REP+1)=2 个 PWM 周期产生一次更新事件 |
| AutoReloadPreload | DISABLE | ARR 立即生效 |
| MasterOutputTrigger | OC4REF | TRGO 输出 = CH4 的 OC4REF |

### 3.2 PWM 频率计算

中心对齐模式，一个完整 PWM 周期 = 2 × (ARR + 1) 个定时器时钟：

```
f_PWM = TIM1CK / (2 × (ARR + 1))
      = 160,000,000 / (2 × 8000)
      = 160,000,000 / 16,000
      = 10,000 Hz = 10 kHz

T_PWM = 1 / f_PWM = 100 µs
半周期 (上计数段) = 50 µs
```

> **注意**: 这与 Simulink 模型步长 `period = 0.0001` (100 µs = 10 kHz) **完全匹配**。

### 3.3 四通道配置汇总

| 通道 | 模式 | 初始 Pulse | 极性 | 硬件连接 | 用途 |
|------|------|-----------|------|----------|------|
| CH1 | PWM1 | 0 | HIGH | PA8 (TIM1_CH1) | U 相上管 |
| CH1N | 互补 | — | HIGH | PB13 (TIM1_CH1N) | U 相下管 |
| CH2 | PWM1 | 0 | HIGH | PA9 (TIM1_CH2) | V 相上管 |
| CH2N | 互补 | — | HIGH | PB14 (TIM1_CH2N) | V 相下管 |
| CH3 | PWM1 | 0 | HIGH | PA10 (TIM1_CH3) | W 相上管 |
| CH3N | 互补 | — | HIGH | PB15 (TIM1_CH3N) | W 相下管 |
| **CH4** | **PWM2** | **7998** | HIGH | 内部 TRGO | **ADC 触发源** |

### 3.4 CH4 触发时序（关键）

CH4 使用 **PWM2 模式**，Pulse = 7998（紧邻 ARR = 7999）：

```
中心对齐三角波 (T = 100 µs):

  7999 ─     ┌──┐     ┌──┐
        │    /    \   /    \
  7998 ───┐/        \/      \┌───
        │  │          │      │
   ...  │  │          │      │
        │  │          │      │
     0 ─┘──┴──────────┴──────┴───→ t
          ↑                    ↑
       OC4REF = 1         OC4REF = 1
     (上升沿触发ADC)    (上升沿触发ADC)
         │                    │
          ←─── 100 µs ──────→
             (一个PWM周期)
```

- OC4REF 脉冲宽度 ≈ 3 个 TIM1 时钟周期 ≈ 18.75 ns
- **每个完整 PWM 周期 (100 µs) 产生 1 个 ADC 触发**
- ADC 注入组触发频率 = PWM 频率 = **10 kHz**
- **FOC_step() 调用频率 = 10 kHz**（与 Simulink 模型完全匹配）

### 3.5 死区时间

CubeMX 配置值为 `120`，通过 HAL 函数 `HAL_TIMEx_ConfigBreakDeadTime()` (`stm32g4xx_hal_tim_ex.c:2278`) 直接写入 TIM1_BDTR 寄存器的 DTG[7:0] 字段：

```c
// stm32g4xx_hal_tim_ex.c:2278
MODIFY_REG(tmpbdtr, TIM_BDTR_DTG, sBreakDeadTimeConfig->DeadTime);  // DeadTime=120 → DTG=0x78
```

**硬件解码** (STM32G4 RM0440, 死区发生器编码表):

| DTG[7:5] 范围 | 死区公式 | 适用范围 |
|-------------|---------|---------|
| **0xx** | DT = DTG[7:0] × t_DTS | 0 ~ 127 × t_DTS |
| 10x | DT = (64 + DTG[5:0]) × t_DTS × 2 | 128 ~ 254 × t_DTS |
| 110 | DT = (32 + DTG[4:0]) × t_DTS × 8 | 256 ~ 504 × t_DTS |
| 111 | DT = (32 + DTG[4:0]) × t_DTS × 16 | 512 ~ 1008 × t_DTS |

DTG = 120 = 0x78 = `0111 1000`，DTG[7:5] = `011` → 落入 **0xx** 范围。

```
t_CK_INT = 1 / TIM1CK  = 1 / 160,000,000 = 6.25 ns    (定时器内核时钟周期)
t_DTS     = 2 × t_CK_INT = 12.5 ns                     (CKD=DIV2, 死区时钟周期)
DT        = 120 × 12.5 ns = 1500 ns = 1.5 µs           ✓
```

| 参数 | 值 | 来源 |
|------|-----|------|
| CubeMX DeadTime | 120 | `tim.c:92` |
| 写入寄存器 | TIM1_BDTR.DTG = 0x78 | HAL `stm32g4xx_hal_tim_ex.c:2278` |
| DTG[7:5] | 011 = 0xx 范围 | 硬件自动解码 |
| CKD (时钟分频) | DIV2 | `tim.c:50` |
| t_DTS | 2 / 160MHz = 12.5 ns | 死区时钟周期 |
| **最终死区时间** | **1.5 µs** | 上下管切换插入的延时 |

## 4. ADC 注入采样详解

### 4.1 ADC1 注入组

| 注入 Rank | 通道 | 采样时间 | 总转换时间 | 信号 | 引脚 |
|-----------|------|----------|-----------|------|------|
| Rank 1 | ADC1_IN3 | 6.5 周期 | 19 周期 = 0.475 µs | A 相电流 (ia) | PA2 |
| Rank 2 | ADC1_IN12 | 6.5 周期 | 19 周期 = 0.475 µs | C 相电流 (ic) | PB1 |

- **注入通道数**: 2
- **触发源**: ADC_EXTERNALTRIGINJEC_T1_CC4 (TIM1 CH4 OC4REF 上升沿)
- **总转换时间**: 2 × 19 = 38 ADC 周期 @ 40MHz ≈ **0.95 µs**
- **中断模式**: 使能 (HAL_ADCEx_InjectedStart_IT)

### 4.2 ADC2 注入组

| 注入 Rank | 通道 | 采样时间 | 总转换时间 | 信号 | 引脚 |
|-----------|------|----------|-----------|------|------|
| Rank 1 | ADC2_IN3 | 6.5 周期 | 19 周期 = 0.475 µs | B 相电流 (ib) | PA6 |

- **注入通道数**: 1
- **触发源**: ADC_EXTERNALTRIGINJEC_T1_CC4 (同一触发源)
- **总转换时间**: 19 ADC 周期 ≈ **0.475 µs**
- **中断模式**: 不使能 (由 ADC1 回调统一处理)

### 4.3 ADC 常规组（低频）

| ADC | 通道 | 采样时间 | 用途 | 触发方式 |
|-----|------|----------|------|---------|
| ADC1 | IN11 | 2.5 周期 = 62.5 ns | 速度电位器 (暂未使用) | 软件触发 |
| ADC2 | IN1 | 2.5 周期 = 62.5 ns | 母线电压 (v_bus) | 软件触发 (主循环轮询) |

## 5. 中断优先级与嵌套

### 5.1 中断向量优先级表

| 优先级 (数字越小越高) | 中断向量 | 用途 |
|------------------------|----------|------|
| **0** | ADC1_2_IRQn | ADC 注入完成 → 读电流 + FOC_step() |
| 0 | TIM4_IRQn | Hall 传感器捕获 (当前未使用) |
| 1 | TIM1_BRK_TIM15_IRQn | TIM1 刹车中断 |
| **2** | USART3_IRQn | 上位机命令接收 |
| 3 | TIM2_IRQn | 通用定时器 (当前未使用) |

### 5.2 嵌套关系

```
主循环 (Thread mode)
  │
  ├── 可被任何中断打断
  │
  └── [中断嵌套] 高优先级可打断低优先级
       │
       ADC1_2_IRQn (优先级 0, 最高)
       │  ├── 不可被 USART3_IRQn 打断
       │  ├── 不可被 TIM2_IRQn 打断
       │  └── 可能被 TIM4_IRQn / TIM1_BRK 抢占 (同优先级, 取决于向量号)
       │
       USART3_IRQn (优先级 2)
       │  ├── 可被 ADC1_2_IRQn 打断
       │  └── 不可被 TIM2_IRQn 打断
       │
       TIM2_IRQn (优先级 3, 最低)
          ├── 可被所有其他中断打断
          └── 不可打断任何中断
```

> **关键结论**: FOC_step() 在优先级 0 的中断上下文中执行，不可被 USART 等低优先级中断打断，保证了 10kHz 实时控制的确定性。

## 6. 完整时序预算

### 6.1 单个 PWM 周期 (100 µs) 内的时间线

```
时间轴 (µs):  0                 50               100
              │←── 上计数 (0→7999) ──→│←── 下计数 (7999→0) ──→│
              │                        │                        │
计数器:       0 → ... → 7998 → 7999   7999 → 7998 → ... → 0    0
                         │      │      │
                         │      │      └── OC4REF↓ (PWM2下降沿)
                         │      └── 计数器顶点
                         └── OC4REF↑ (PWM2上升沿 → ADC触发)
                              │
                              ├─ 0 µs: TRGO 发出触发脉冲
                              ├─ ~0.1 µs: ADC1/ADC2 同步开始注入转换
                              ├─ ~0.48 µs: ADC2 完成 (B相)
                              ├─ ~0.95 µs: ADC1 完成 (A相 + C相),
                              │            JEOC 标志置位 → 进入 ADC1_2_IRQn
                              │
                              ├─ [中断服务例程 ~8-15 µs]
                              │   ├─ 0.5 µs: ISR 入口, 读取 ADC 值
                              │   ├─ 0.2 µs: offset 修正, A转换
                              │   ├─ 6-12 µs: **FOC_step() 核心计算**
                              │   ├─ 0.1 µs: 写入 TIM1->CCR1/2/3
                              │   └─ 0.5 µs: (条件) 启动 VOFA DMA TX
                              ├─ ISR 退出
                              │
                              └─ CCR 影子寄存器在下一更新事件加载
                                 (因 REP=1, 每 2 周期更新一次, 即每 200 µs)

剩余时间 (100 - 15 = ~85 µs): CPU 可执行主循环代码或低优先级中断
```

### 6.2 FOC_step() 内部计算分解

基于 Cortex-M4F @ 160 MHz 的估算（160 MHz 下 FPU 单精度乘加 ≈ 1 周期 = 6.25 ns）：

| 步骤 | 操作 | 三角函数 | 浮点运算 | 估算耗时 |
|------|------|---------|---------|---------|
| 1. Clarke 变换 | ia,ib,ic → iα,iβ | 0 | ~5 | <0.2 µs |
| 2. 状态机 | 5 状态启动流程 | 0 | 逻辑判断 | <0.2 µs |
| 3. 开环/闭环选择 | Switch-case | 0 | ~10 | <0.2 µs |
| 4. sinf + cosf | 计算 sin(θ), cos(θ) | **2** | 0 | **2~3 µs** |
| 5. Park 变换 | iα,iβ → id,iq | 0 | ~6 | <0.2 µs |
| 6. Id PI 调节器 | 含抗饱和积分 | 0 | ~12 | <0.3 µs |
| 7. Iq PI 调节器 | 含抗饱和积分 | 0 | ~12 | <0.3 µs |
| 8. 电压限幅 | Vd,Vq 钳位 ±12.47V | 0 | ~8 | <0.2 µs |
| 9. 逆 Park 变换 | Vd,Vq → Vα,Vβ | 0 | ~6 | <0.2 µs |
| 10. SVPWM | Vα,Vβ → Tcmp1/2/3 | 0 | ~15 | <0.3 µs |
| 11. 磁链观测器 | 非线性观测器 | 0 | ~20 | <0.5 µs |
| 12. PLL 锁相环 | 含 sinf/cosf | **2** | ~10 | **2~3 µs** |
| 13. 速度环 (1/10) | 仅 TID[1]==0 时执行 | 0 | ~15 | <0.3 µs (均摊 <0.03) |
| 14. 积分器更新 | 各状态变量更新 | 0 | ~15 | <0.3 µs |
| **合计** | | **4** | **~110** | **~6-12 µs** |

> **注意**: 三角函数 `sinf()`/`cosf()` 是 FOC_step() 中耗时最长的单步操作。使用 CMSIS-DSP 的 `arm_sin_cos_f32()` 可同时计算 sin 和 cos，约节省 30~50% 的三角函数时间。

### 6.3 CPU 负载估算

```
FOC_step() 执行时间: ~8 µs (取典型值)
ADC 中断总时间:     ~10 µs (含 ISR 进出 + FOC)
PWM 周期:           100 µs

CPU 负载 (FOC + ADC): 10 / 100 = 10.0%
剩余可用于主循环:      90.0%
```

> 160 MHz 下 FOC 计算非常从容，CPU 负载仅 10%。即使在最坏情况下 (含速度环执行，~12 µs) 也仅 12%。

## 7. 时序嵌套全景图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         系统上电初始化 (~500ms)                          │
│  HAL_Init → SystemClock_Config → FOC_initialize → 外设初始化             │
│  → 启动 TIM1_CH4 (仅触发) → ADC 注入采样开始                             │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                     Offset 标定阶段 (~102.4 ms)                         │
│  每次 ADC 中断: 累加 ia/ib/ic ADC 值, 不执行 FOC                        │
│  累计 1024 次后取平均 → current_offset_ready = 1                        │
│  1024 / 10000 Hz = 102.4 ms                                             │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼ (等待上位机 RUN=1)
┌─────────────────────────────────────────────────────────────────────────┐
│                     FOC 正常运行 (无限循环)                              │
│                                                                         │
│  ┌─── TIM1 中心对齐 PWM (f=10 kHz, T=100 µs) ──────────────────────┐  │
│  │                                                                    │  │
│  │  每次 PWM 周期 OC4REF↑ 触发:                                      │  │
│  │    ADC1/2 注入转换 (~1 µs) → ADC1_2_IRQn (优先级 0)              │  │
│  │      ├── 读 ia/ib/ic ADC                                          │  │
│  │      ├── offset 修正 → A 转换                                     │  │
│  │      ├── FOC_step() (~8 µs)   ←── 核心计算                        │  │
│  │      │    ├── Clarke/Park 变换                                    │  │
│  │      │    ├── PI 电流环 (Id,Iq)                                   │  │
│  │      │    ├── SVPWM                                               │  │
│  │      │    ├── 磁链观测器 + PLL (含 sinf/cosf ×4)                  │  │
│  │      │    └── 每 10 次: 速度环 PI → 速度环 = 1 kHz               │  │
│  │      ├── 写 CCR1/2/3 (影子寄存器, 下次更新事件生效)               │  │
│  │      └── (条件) DMA TX VOFA 帧                                    │  │
│  │                                                                    │  │
│  │  ┌─ ISR 返回, 剩余 ~90 µs 给主循环 ──────────────────────────┐   │  │
│  │  │  主循环 (每 10ms 一轮):                                      │   │  │
│  │  │    HostCommand_ProcessPending() → 解析上位机命令              │   │  │
│  │  │    读 v_bus (ADC2 regular 轮询)                              │   │  │
│  │  │    启动/停机状态机                                           │   │  │
│  │  │    HAL_Delay(10)                                             │   │  │
│  │  └──────────────────────────────────────────────────────────────┘   │  │
│  └────────────────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────────────┘

嵌套层次:
  L0: 主循环 (Thread mode, 最低优先级)
   ├── 可被 L1/L2 任意打断
   │
  L1: USART3_IRQn (优先级 2)
   ├── 可被 L2 打断
   └── 打断 L0
   │
  L2: ADC1_2_IRQn (优先级 0, 最高)
   ├── 不可被 L0/L1 打断
   └── 其中包含 FOC_step() (计算核心)
```

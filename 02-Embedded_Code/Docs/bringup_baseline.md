# MyFOC_NFlux 当前稳定基线

日期：2026-06-14

## 当前状态

- 工程已经可以在 STM32G431 实板上运行传感器less PMSM FOC。
- Simulink 生成的 FOC 代码放在 `Matlab/`，由 `Core/Src/main.c` 调用。
- 上电后先做三相电流 offset 标定，标定完成后自动启动三相 PWM。
- 速度给定当前固定为 `RefSpeed = 600.0f`，不使用速度电位器。
- 母线电压由 ADC 采样并滤波，当前控制入口使用 `vbus_lpf`。
- USART3 通过 CH340 Type-C 接电脑，VOFA JustFloat 波形已经验证正常。
- 用户实测反馈：当前波形和运行效果都很好。

## 关键实现点

- `TIM1_CH4` 只作为 ADC injected 触发源，先启动它完成电流 offset 标定。
- 三相 PWM 输出 `TIM1_CH1/2/3` 和互补输出等 offset 完成后再启动，避免上电抽动。
- ADC injected 回调中读取三相电流、扣 offset、换算成安培，然后执行 `FOC_step()`。
- `FOC_step()` 输出 `Tcmp1/Tcmp2/Tcmp3`，直接刷新 TIM1 三相 PWM 比较值。
- VOFA 上传格式为 6 个 `float` 加 JustFloat 帧尾 `00 00 80 7F`。

## 当前 VOFA/上位机遥测通道

| 通道 | 含义 |
| --- | --- |
| 0 | `ia`，A 相电流 |
| 1 | `ib`，B 相电流 |
| 2 | `ic`，C 相电流 |
| 3 | `FluxTheta`，观测磁链角 |
| 4 | `FluxWm`，观测速度 |
| 5 | `RefSpeed`，速度参考 |
| 6 | `rtU.v_bus`，进入控制模型的母线电压 |
| 7 | `vbus_raw`，ADC 换算后的原始母线电压观察值 |

## 下一阶段

下一步先做 PC 上位机，而不是速度电位器或参数辨识。

上位机第一版目标：

- 串口连接/断开。
- 实时显示当前 6 个遥测量。
- 图形曲线显示，功能上替代 VOFA 的基础观察窗口。
- 支持通道选择、鼠标缩放/拖拽、Y 轴自动缩放、跟随最新数据。
- 支持 2x2 曲线布局、三相电流同窗显示、图例点击隐藏/恢复曲线、深色主题。
- 下发启动/停止命令。
- 下发速度给定。
- 保存 CSV 数据，方便后面做阶跃测试和参数辨识。

参数辨识放在上位机控制和数据记录稳定之后再做。

## 上位机目录

- 上位机源码放在 `PC_Tool/`。
- 运行入口是 `PC_Tool/run.bat`。
- 串口协议说明见 `PC_Tool/protocol.md`。
- 图形界面使用 `PySide6 + pyqtgraph`，适合实时曲线、缩放和平移。

## 上位机控制版行为

- 稳定基线 commit 保留了上电自动运行版本。
- 当前上位机控制版上电后只做电流 offset 标定，不会自动启动电机。
- 上位机发送 `RUN=1` 后，主循环才会启动三相 PWM 并让 FOC 模型运行。
- 上位机发送 `RUN=0` 后，固件会关闭三相 PWM 输出。
